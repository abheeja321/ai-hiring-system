# Zecpath AI Screening System: Architecture

## Overview

The Zecpath AI Screening System is a comprehensive evaluation platform designed to automate the screening of candidates based on their resumes and job descriptions. The system leverages modern Natural Language Processing (NLP) to extract structured profiles, compare candidates against job requirements, and run multi-faceted evaluations including ATS scoring, AI behavioral analysis, and interview intelligence.

## Core Components

### 1. Web Application & API
- **Django Backend:** Powers the core infrastructure, managing the database, business logic, and integrations.
- **REST API (DRF):** Provides endpoints to ingest job descriptions, candidate resumes, and trigger the hiring pipeline programmatically (`/api/v1/runs/run_pipeline/`).
- **Dashboard UI:** A web interface built with Django Templates to manually run the pipeline, view evaluation breakdowns, and explore hiring history.

### 2. AI Engine (`zecpath_hiring.ai`)
The intelligence layer is divided into specialized modules:

#### Parsers (`ai.parsers`)
- `jd_parser`: Extracts structured data (skills, experience, requirements) from raw job descriptions.
- `resume_parser`: Extracts structured profiles (education, experience, technical skills) from candidate resumes.

#### Evaluators (`ai.ats_engine`, `ai.screening_ai`, `ai.interview_ai`, `ai.behavior_ai`)
- **ATS Engine:** Calculates hard matches between resume skills and job requirements.
- **Screening AI:** Determines basic eligibility and filters candidates based on core criteria.
- **Interview Intelligence:** Simulates and scores potential technical interview responses based on the candidate's background.
- **Behavior AI:** Analyzes soft skills, cultural fit, and behavioral traits from the resume text and historical data.

#### Decision Maker (`ai.decision_ai`)
- Aggregates scores from all modules and formulates a final decision (e.g., `HIRE`, `REVIEW`, `REJECT`), generating a comprehensive explanation for the human recruiter.

## Database Schema

The system uses standard relational models stored in SQLite (or PostgreSQL in production):

- `JobProfile`: Stores raw and structured job requirements.
- `CandidateProfile`: Stores candidate details, raw resume text, and structured profile data.
- `HiringRun`: Represents a single evaluation pipeline execution, linking a candidate and job, and storing all component scores and the final decision.
- `AIArtifact`: Auditable record of JSON payloads and responses generated by the AI models during a pipeline run.
- `ScreeningInteraction`: Logs individual QA or chat interactions if the candidate goes through an interactive screening session.

## Data Flow Pipeline

1. **Ingestion**: Raw JD and Resume text are submitted via API or Web UI.
2. **Parsing**: The text is parsed into structured JSON schemas.
3. **Evaluation**:
   - ATS Engine scores hard skills.
   - Screening evaluates basic eligibility.
   - Behavior & Interview models assess soft skills and technical depth.
4. **Aggregation**: The Decision AI aggregates the module scores.
5. **Storage & Audit**: Results are saved to `HiringRun`, and raw inputs/outputs are logged in `AIArtifact`.
6. **Response**: The final evaluation report is returned to the user or API caller.
