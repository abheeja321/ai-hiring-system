# Zecpath AI Screening API Reference

The Zecpath API allows external systems to programmatically submit candidates and job descriptions, run the AI screening pipeline, and retrieve evaluation results. 

## Base URL
`/api/v1/`

## Authentication
*(Currently relies on standard Django session authentication if implemented. In production, configure Token Authentication or JWT via Django REST Framework.)*

---

## Endpoints

### 1. Hiring Runs
Manage and trigger AI evaluation pipelines.

#### `GET /api/v1/runs/`
List all historical hiring pipeline runs.
- **Response:** `200 OK` (Array of HiringRun objects)

#### `POST /api/v1/runs/run_pipeline/`
Triggers the full AI evaluation pipeline for a given candidate and job description. This is the primary entry point for the AI Screening System.

**Request Payload:**
```json
{
  "candidate_name": "Jane Doe",
  "resume_text": "Experienced Software Engineer with 5 years in Python and Django...",
  "job_title": "Backend Developer",
  "job_description": "We are looking for a backend developer proficient in Python, Django, and REST APIs..."
}
```

**Response Payload (`201 Created`):**
```json
{
  "id": 101,
  "candidate": 50,
  "job": 20,
  "ats_score": 85.5,
  "screening_score": 90.0,
  "interview_score": 80.0,
  "behavior_score": 88.0,
  "final_score": 86.5,
  "decision": "HIRE",
  "explanation": {
     "ats": { ... },
     "behavior": { ... },
     "decision": { ... }
  },
  "created_at": "2023-10-27T10:00:00Z"
}
```

### 2. Job Profiles
Manage job descriptions.

#### `GET /api/v1/jobs/`
List all stored job profiles.

#### `GET /api/v1/jobs/{id}/`
Retrieve a specific job profile, including its AI-structured representation.

### 3. Candidate Profiles
Manage candidate resumes and profiles.

#### `GET /api/v1/candidates/`
List all stored candidates.

#### `GET /api/v1/candidates/{id}/`
Retrieve a specific candidate, including their AI-structured resume data.

### 4. AI Artifacts
Audit logs for internal AI reasoning.

#### `GET /api/v1/artifacts/`
List all JSON payloads generated by the AI models during pipeline executions. Useful for debugging model performance and tracking prompt/response histories.

### 5. Interactions
Logs for interactive screening sessions (if enabled).

#### `GET /api/v1/interactions/`
Retrieve transcripts and confidence scores of candidate QA sessions.
